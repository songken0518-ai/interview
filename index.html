A

開場與自我介紹

Opening
HR ＋ 主管
前 60 秒決定第一印象。目標：清楚、有自信、每句都往「我適合這職位」收。
Q請先簡單自我介紹。點擊展開參考答法 →▾
考官在測什麼30–60 秒內能不能講清楚「你是誰、為什麼在這、為什麼適合」，順便看你的口條和自信。
怎麼答四段式，一口氣 60 秒講完：
「我原本在電子業，打下硬底子；後來當程式老師，把一件事練到位——把複雜的東西拆解到別人能懂。這幾年我自己動手做了完整的 LLM 應用：多模型容錯系統、RAG、API 整合。做完我很確定，比起教別人寫，我更想自己蓋這些系統，所以來應徵這個職位。」
可能追問
  • 那你為什麼不繼續教？
  • 你做的系統可以多講一點嗎？
別這樣答
  • 從求學經歷開始流水帳
  • 講超過 90 秒
  • 只報學經歷、不連結到這個職位
Q你怎麼會想投我們公司這個職位？點擊展開參考答法 →▾
考官在測什麼你有沒有做功課、懂不懂這職位在幹嘛、動機真不真。
怎麼答三塊：職位吻合（我做的 LLM 應用就是這職位的日常）＋對公司的具體理解（AI 行銷生態鏈、串接數百家媒體與電商、數據自動引擎、朝國際市場）＋我能帶來什麼（把商業需求翻成能落地又穩定的系統）。
別這樣答
  • 空話『貴公司很有前景』
  • 講不出公司實際在做什麼
B

轉職動機（最核心）

Why the Switch
主管 必問
這是你的最大關卡，也是最大機會。核心策略：把轉職講成「累積式的刻意選擇」，不是「逃跑」或「趕流行」。
Q為什麼從老師 / 電子業轉來做 AI 工程？點擊展開參考答法 →▾
考官在測什麼動機穩不穩、會不會做不久、是不是一時衝動或跟風。
怎麼答「我的路線一直往『把技術做出來、講清楚』靠。電子業給我硬底子，教學讓我練成拆解複雜問題的能力。但我不想只停在教，所以我自己上手做了完整的 LLM 應用系統——不是看教學，是從零踩坑做到能跑。做完我確定這是我想全職投入的方向。我把接下來這幾年定位成扎實累積 AI 工程實作的階段。」
可能追問
  • 教書不好嗎？
  • 你確定不會做一做又想換？
  • 為什麼是現在？
別這樣答
  • 抱怨教職 / 前東家
  • 說『因為 AI 很紅』
  • 語氣像『先來試試看』
Q你會不會做不久？（常包在別的問題裡問）點擊展開參考答法 →▾
考官在測什麼留任風險。這是主管心裡真正在算的帳。
怎麼答用證據而不是保證回答：「我不是隨便投投看——我自建了一整套作品集來確認方向、補足資歷，這本身就證明我是認真的。我要的是深耕、把系統做扎實，不是跳板式亂換。」
別這樣答
  • 空口說『我會待很久』卻沒依據
  • 閃避或顧左右而言他
Q你沒有正式的軟體工程師資歷，怎麼補足？點擊展開參考答法 →▾
考官在測什麼自我認知 ＋ 學習力。誠實度也在被看。
怎麼答坦白 ＋ 補強：「相關全職資歷我約一年，比較短，這我清楚。但我用作品把實作補起來，而且教學練出的快速學習與拆解能力，讓我上手新技術很快。【你補：舉一個你自學新技術、短時間做出東西的具體例子】」
別這樣答
  • 假裝自己很資深
  • 過度自貶、顯得沒自信
C

對公司與職位的理解

Company & Role
HR ＋ 主管
考『你有沒有搞懂應用工程師在做什麼』。應用工程師 ≠ 研究員，重點是把東西做出來、對接需求。
Q你對我們公司了解多少？點擊展開參考答法 →▾
考官在測什麼做功課的程度、是不是真的想來。
怎麼答用自己的話講、別背稿：台灣總部＋越南分部、串接數百家媒體與電商、主打數據自動引擎與AI 行銷生態鏈、正朝國際市場擴張。再接一句「這種把 AI 落地成行銷產品的方向，正好是我想投入的」。
別這樣答
  • 只會說『你們做 AI』
  • 把官網文案硬背出來
Q你覺得 LLM 應用工程師在我們公司負責什麼？點擊展開參考答法 →▾
考官在測什麼你懂不懂這角色的定位——落地、對接、跨部門。
怎麼答「把 LLM 能力落地成行銷產品：文案生成、受眾溝通、內容自動化。技術上要串接資料（RAG）、接工具（function calling / MCP）、確保穩定與輸出品質，還要跟行銷、PM 對接，把模糊的商業需求翻成能跑的系統。」
別這樣答
  • 把自己講成只想埋頭研究模型的人
D

作品深挖（技術主戰場）

Portfolio Deep-Dive
主管 重兵
主管 60 分鐘大半會花在這。他們要確認：是不是你真的做的、有多深、能不能講清楚取捨。這區的【你補】最關鍵，一定要填真實細節。
Q挑一個你最有成就感的作品，從頭講一遍。（主推 failover 評分系統）點擊展開參考答法 →▾
考官在測什麼作品的真實性與深度；你講故事的結構力。
怎麼答用做什麼 / 解決什麼 / 關鍵決策三段：
做什麼：一套自動評分系統，串接多個 LLM，主模型失敗或逾時就自動切備援。
解決什麼：單一模型會限流、逾時、掛掉，直接讓服務中斷；failover 保住可用性與輸出穩定。
關鍵決策：【你補：你怎麼判定主模型失敗（逾時門檻？連續錯誤？）／retry 與 backoff 怎麼設／不同模型的評分怎麼保持一致】
可能追問
  • 為什麼要 failover，不直接用最強的單一模型？
  • 你怎麼知道主模型『失敗』了？
  • 多模型的成本你怎麼權衡？
別這樣答
  • 只講功能不講技術取捨
  • 細節答不出來（代表不是你做的）
Q你的 RAG 系統，檢索不準的時候你怎麼調？點擊展開參考答法 →▾
考官在測什麼有沒有實戰調校經驗，不是只看過教學。
怎麼答由淺到深講手段：chunk size / overlap → 換更強 embedding → hybrid（向量＋關鍵字）→ rerank → metadata 過濾。然後接真實案例：【你補：你實際遇到什麼檢索不準的狀況、最後靠哪一招解掉】
可能追問
  • 為什麼那一招有效？
  • 你怎麼評估檢索有沒有變準？
別這樣答
  • 只背名詞、講不出你實際調過哪個
Q如果讓你重做這個系統，你會改什麼？點擊展開參考答法 →▾
考官在測什麼反思力與工程成熟度——資淺的人常答不出來。
怎麼答誠實講一個真實的不足或取捨＋你會怎麼改。【你補：例如當初評估流程太手動、沒做好監控、chunking 用了太粗的切法……挑一個真的，講你的改法】
別這樣答
  • 說『沒什麼好改的，都很完美』
Q你遇過最難的技術問題是什麼？怎麼解的？點擊展開參考答法 →▾
考官在測什麼解決問題的方法——特別是根因排查能力。
怎麼答用 STAR（情境→任務→行動→結果）講一個真的。刻意帶到你的工作習慣：不堆補丁、追根因、交付前自己先跑一輪驗證——這正是他們要的工程紀律。【你補：真實案例】
別這樣答
  • 舉一個其實不難的問題
  • 把功勞含糊帶過或甩鍋
E

技術理解題（口說版）

Verbal Tech
主管
面試是用『講』的，30 秒白話講懂才算贏，不能像寫考卷。考官要的是『你是真的理解』。深入細節可翻你的筆試複習網頁。
Q用白話講，什麼是 RAG？為什麼要用它？點擊展開參考答法 →▾
怎麼答「先找資料、再作答。把私有文件切塊、轉成向量存起來；使用者一問，先檢索最相關的片段，連同問題一起餵給模型。好處是模型有本可讀，不會憑記憶瞎掰，答案還能溯源，資料更新也不用重訓。」
Qtemperature 怎麼設？行銷文案跟固定格式輸出差在哪？點擊展開參考答法 →▾
怎麼答「temperature 控制隨機性。行銷文案要創意，抓 0.7–0.9；要固定 JSON 或欄位格式，壓到 0.2–0.4 保穩定。原則是跟 top_p 只調一個，別兩個一起亂動。」
QLLM 會幻覺，你怎麼降低？點擊展開參考答法 →▾
怎麼答「主要靠 RAG 接地——給它真資料並要求『只根據提供內容作答』；再加上允許它說不知道、要求附來源、降 temperature、事後驗證。」
Qfunction calling 和 MCP 是什麼？點擊展開參考答法 →▾
怎麼答「function calling 是讓模型輸出一個『呼叫某工具＋參數』的請求，由我的程式去執行、再把結果回給模型，讓它能動手做事、拿即時資料。MCP 是把這些工具用統一標準接上來的協定，像通用轉接頭，把 M×N 的整合問題變成 M＋N。」
F

情境 / 系統設計

System Design
主管
貼合禾多業務的申論題最能展現『你真的做過』。用一條端到端流水線回答，展現系統思維。
Q要你做一個「自動生成商品行銷文案」系統，你會怎麼設計？點擊展開參考答法 →▾
考官在測什麼系統思維、能不能落地、有沒有想到品質與成本。
怎麼答一條線講完：輸入（商品資料、客群、渠道）→ RAG 抓品牌調性與過去高成效文案當範例 → 組 prompt（模板＋few-shot＋規則）→ 生成＋failover（主模型掛了切備援，你的強項）→ 自動品管（禁用詞、字數、調性分類器、對照商品資料查事實）→ 人工審核佇列 → A/B 測試比 CTR，勝出的回灌成新範例，形成正循環。
可能追問
  • 成本和延遲怎麼優化？
  • 怎麼確保不同品牌的調性不跑掉？
別這樣答
  • 只講『叫模型生成』一句話帶過
Q生成內容出現錯誤或不當內容，怎麼防？點擊展開參考答法 →▾
怎麼答三層防線：生成前（system prompt 鎖規則、低 temperature）／生成後自動（禁用詞、格式、調性分類器、事實比對）／生成後人工＋資料（邊界案例人審、用線上成效當長期品質訊號）。
G

行為題（軟實力）

Behavioral
HR 主場
全部用 STAR 結構（情境→任務→行動→結果），答案要真。這區你有優勢——教學與跨部門經驗是天然素材。
Q講一次你跟別人意見不合、怎麼處理的經驗。點擊展開參考答法 →▾
考官在測什麼協作與情緒成熟度。應用工程師要一直跟行銷、PM 對接。
怎麼答STAR ＋ 帶到你的原則：先弄懂對方真正要解決的問題、對事不對人、用資料 / 驗證結果說話。【你補：一個真實案例，教學或工作場合都可以】
別這樣答
  • 把對方講得很糟、自己全對
  • 說『我沒跟人不合過』（顯得不真）
Q你的缺點是什麼？點擊展開參考答法 →▾
考官在測什麼自我覺察與誠實。假缺點一聽就破。
怎麼答講真缺點＋正在改的行動＋不傷核心職能。可用的方向：資深架構 / 大型系統經驗還在補——然後講你怎麼補（讀原始碼、做專案、找 code review）。【你補：挑一個你真的在改的】
別這樣答
  • 『我太追求完美 / 我太投入工作』這種假缺點
  • 講一個會嚇跑面試官的致命缺點
Q遇到壓力大、期限很緊怎麼辦？點擊展開參考答法 →▾
怎麼答「先拆優先序、早一點把風險講出來、交付前先自己驗證一輪再說完成。我不喜歡壓線才發現問題。」（這正好是你既有的工作原則，講起來自然。）
Q講一次你失敗的經驗。點擊展開參考答法 →▾
考官在測什麼能不能誠實面對失敗並從中學習。
怎麼答STAR：真實失敗 → 你當下怎麼處理 → 學到什麼、後來怎麼改。重點放在成長，不是災難本身。【你補】
別這樣答
  • 講假失敗
  • 把責任推給別人或環境
H

職涯規劃

Career Plan
主管 ＋ HR
⚠️ 這區有地雷。你心裡的『2–3 年跳板』規劃，對雇主要翻譯成『深耕』的說法。
Q你未來 3–5 年的規劃？點擊展開參考答法 →▾
考官在測什麼你會不會來鍍個金就走。
怎麼答「這幾年我想深耕 AI 應用工程，從能獨立完成模組，到能獨立扛起一整個系統。我希望在一個有真實產品、真實資料量的地方累積，這也是我看重你們的原因。」
別這樣答
  • 提『想創業』『想出國』『想自己接案』——會讓對方覺得你要閃
  • 說『還沒想那麼遠』
Q為什麼你把這段定位成 2–3 年？（若你不小心透露了跳板規劃）點擊展開參考答法 →▾
考官在測什麼承諾度。這題其實是陷阱。
怎麼答⚠️ 最佳解是一開始就別讓『跳板』這個框架進到對話。萬一被追問時間，就把重點轉到成果而非年限：「我不是用年限在算，而是想把系統真正做扎實、看到產品成效——那需要的是投入與累積，不是設一個離開的鬧鐘。」
別這樣答
  • 老實說『我把這裡當跳板』
  • 講『做兩三年我就要往上跳槽』
I

HR 行政題

Logistics
HR
Step 2 HR 面談會問這些。事前把數字想好，別現場才算。
Q你的期望薪資？點擊展開參考答法 →▾
考官在測什麼你懂不懂行情、好不好談。
怎麼答給區間＋依據，保留彈性：「依市場行情與我作品能帶來的價值，我的期待落在【你補：先查 104 / 相近職缺行情，訂一個合理區間】，但我更在意職級定位跟整體發展，細節可以談。」再反問薪資結構與職級。
別這樣答
  • 開一個沒依據的天價或地板價
  • 說『都可以、你們決定』（自貶議價空間）
Q最快什麼時候可以到職？點擊展開參考答法 →▾
怎麼答據實回答。若教職有交接 / 學期因素，講清楚時程，展現你負責、不落跑的態度。【你補：你的實際可到職時間】
Q你有什麼想問我們的？（一定會問，絕不能說沒有）點擊展開參考答法 →▾
考官在測什麼你的思考深度與投入度。答『沒有』＝大扣分。
怎麼答→ 見下方「你該反問的好問題」，準備 3–4 個，現場挑 2 個問。
★

你該反問的好問題

Questions to Ask Them
必備 3–4 個
面試尾聲問『你有什麼想問的』時用。答『沒有』＝大扣分。
  1. 這個職位進來的前三個月，你們最希望我做出什麼具體成果？
